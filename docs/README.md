# Documentation

Central documentation hub for EmpireAI—architecture, decisions, guides, and onboarding.

## Responsibilities

- System architecture and data flow diagrams
- ADRs (Architecture Decision Records)
- Developer setup and contribution guides
- User-facing help and feature documentation
- Runbooks for operations and incident response

## Planned Structure

```
docs/
├── architecture/  # System design, diagrams, and ADRs
├── api/           # Human-readable API guides (specs live in ../api/)
└── guides/        # How-to guides for developers and operators
```

## Conventions

- Keep docs close to truth—update when interfaces or behavior change
- Prefer markdown; link to OpenAPI and code where appropriate
- Date significant decisions in ADRs

## Status

Scaffold only. Content to be added as the system is built.
