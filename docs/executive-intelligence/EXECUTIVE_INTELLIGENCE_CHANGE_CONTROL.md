## Governance rules

Once an EI document reaches **Approved Version 1.0**:

It shall not be silently modified.

Changes require:

- Version increment
- Decision history
- Approval history
- Reason for amendment

No executive doctrine shall be altered without constitutional traceability.

---

## Amendment workflow (documentation)

| Step | Requirement |
|------|-------------|
| 1 | Identify EI document and current version |
| 2 | Record reason for amendment |
| 3 | Increment version (e.g. v1.0 → v1.1 or v2.0) |
| 4 | Record decision history (ADR or Executive Intelligence decision entry) |
| 5 | Record approval history (Grand King / designated authority) |
| 6 | Update document header fields: Version, Effective Date, Supersedes, Next Version |
| 7 | Sync [EI_INDEX.md](./EI_INDEX.md) status row |

---

## Prohibited

- Silent edits to Approved v1.0 EI documents
- Doctrine changes without version increment
- Doctrine changes without approval history
- Runtime or code changes masquerading as EI documentation updates (EI governs reasoning; implementation follows separate REAL / engineering missions)
