# EmpireAI — Repository First Doctrine

**Canonical label:** Repository First Doctrine  
**Status:** ✅ Permanent repository governance rule  
**Registered:** BL-B Item 010 (2026-06-29)  
**Canonical owners:** Repository Governance · Pillow Architecture

---

## 1. Doctrine

The **repository is the permanent memory** of EmpireAI.

Conversation exists **only** to produce approved repository knowledge.

---

## 2. Authority hierarchy

| Source | Authority |
|---|---|
| Canonical repository artifacts | **Authoritative** |
| Conversation / chat history | Temporary — **not authoritative** after sync |

If conflict exists between repository artifacts and remembered conversations, the **repository wins** unless Grand King **explicitly** approves otherwise.

---

## 3. Pillow obligation

Pillow shall continuously synchronize approved knowledge into **canonical repository owners**.

Pillow shall **never** treat conversation history as authoritative after repository synchronization.

---

## 4. Cursor obligation

Cursor shall never invent repository owners.

If no appropriate owner exists:

1. Report the missing owner.  
2. Recommend the correct canonical owner.  
3. Await Grand King's approval before creating a new owner.
