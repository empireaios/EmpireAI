# EmpireAI Journey

**Purpose:**
This is the master index of EmpireAI.
It exists so Grand King, Pillow, Cursor, and future AI workers can understand exactly what has been built, what each label means, and where EmpireAI is in the journey.

This index is documentation only. No runtime files were modified to produce it. Every row below is sourced from the existing repository (doctrine catalogs, ledgers, audit documents, the UX contract, generated files, and Grand King-approved governance decisions). Labels that were requested but have no repository definition are **not** invented here — they are recorded in `JOURNEY_AUDIT.md`.

**Living artifact (per BL-A):** Journey reflects implemented repository reality and Grand King-approved governance. It is never silently rewritten — every structural change is logged in `JOURNEY_AUDIT.md`. Repository continuity (these artifacts) is EmpireAI's permanent memory; conversation history is temporary. As of BL-A, the names **MS-A**, **MS-B**, **Pillow**, and **BL-A** are canonically defined here (see the Governance & Milestones phase) by Grand King decision, not invented.

**Per-release sequence (ROUTE 02 / ADR-020):** every Backlog Release must run **Audit Repository → Refresh JOURNEY.md → Refresh JOURNEY_AUDIT.md → Repository Difference Report → Synchronization Report**, routing each approved decision to its **canonical owner** (Source → Owner → Repository Action → Validation; never synchronize generic ideas). Future missions must first determine "does this change Journey?" before implementation. **BL-B is closed (2026-06-29).** **BL-C is ACTIVE (2026-06-29)** — continuous improvement constitution and enhancement registers.

**REAL namespace governance (ADR-044):** canonical REAL-### ownership, duplicate-ID prohibition, and runtime-truth precedence are permanently defined in `docs/governance/ADR-044-REAL-NAMESPACE-CANONICALIZATION.md`. Commerce/runtime REAL-003/004/005 and REAL-055 remain indexed with ⚠️ where foundation or blueprint legacy conflicts persist; **no renumbering** until post-V1 governed mission.

**Commercial architecture transition (ADR-045):** REAL-002B completes foundational commercial **integration** (architecture); post-V1 strategic focus shifts to **Commercial Intelligence** (Roadmap Layer 3). See `docs/governance/COMMERCIAL_INTEGRATION_TO_INTELLIGENCE_TRANSITION.md`.

**Executive Cognitive Pipelines (ADR-046):** Pillow Executive Intelligence (Layer 2) PEI missions organised into pipelines A–E for Master Plan planning. See `docs/governance/EXECUTIVE_COGNITIVE_PIPELINES.md`. Constitution and PEI identifiers unchanged.
